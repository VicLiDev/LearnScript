# 正则表达式

这里讲的内容应该只针对bash，未必适用于其他shell

---

> 适用范围：`[[ string =~ regex ]]`
> **不是 grep / sed / awk 的正则**

---

> **Bash 正则 = 条件判断 + 捕获工具
> 不是文本处理工具**

---

## 基础语法

---

**Bash 正则的使用位置**

Bash 正则 **只能在 `[[ ... =~ ... ]]` 里使用**
```bash
[[ STRING =~ REGEX ]]
```

成功 / 失败
| 结果           | 含义     |
| -------------- | -------- |
| true (返回 0)  | 匹配成功 |
| false (返回 1) | 匹配失败 |

---

**最重要的规则**

正则 **不要加引号**
```bash
# 错误
[[ "$s" =~ "abc.*" ]]

# 正确
[[ "$s" =~ abc.* ]]
```
> 一旦加引号，Bash 会把正则当成普通字符串

---

**BASH_REMATCH**

当正则匹配成功后，Bash 会自动生成数组：
```bash
BASH_REMATCH
```

数组含义
| 索引  | 内容               |
| ----- | ------------------ |
| `[0]` | 整个匹配到的字符串 |
| `[1]` | 第一个 `()`        |
| `[2]` | 第二个 `()`        |
| ...   | 依次类推           |

**示例**
```bash
s="Frame123"
[[ "$s" =~ ([A-Za-z]+)([0-9]+) ]]

echo ${BASH_REMATCH[0]}   # Frame123
echo ${BASH_REMATCH[1]}   # Frame
echo ${BASH_REMATCH[2]}   # 123
```

## 基础正则符号（Bash 支持）

字符匹配
| 写法    | 含义          |
| ------- | ------------- |
| `a`     | 匹配字母 a    |
| `.`     | 任意单个字符  |
| `[abc]` | a 或 b 或 c   |
| `[a-z]` | 任意小写字母  |
| `[A-Z]` | 任意大写字母  |
| `[0-9]` | 任意数字      |

---

次数匹配（量词）
| 符号 | 含义         |
| ---- | ------------ |
| `*`  | 0 次或多次   |
| `+`  | 1 次或多次   |
| `?`  | 0 次或 1 次  |

```bash
[[ "aaa" =~ a+ ]]   # true
[[ "" =~ a* ]]      # true
[[ "a" =~ a? ]]     # true
```

---

分组 `()`
```bash
(abc)
```
* 把多个字符当成一个整体
* **必须用 () 才能进 BASH_REMATCH**

---

或 `|`
```bash
cat|dog
```
匹配 `cat` 或 `dog`

---

锚点（位置控制）
| 写法  | 含义       |
| ----- | ---------- |
| `^`   | 字符串开头 |
| `$`   | 字符串结尾 |
```bash
[[ "abc123" =~ ^abc ]]   # true
[[ "abc123" =~ 123$ ]]   # true
```

---

## 常用模式总结

数字
```regex
[0-9]+
```

字母
```regex
[A-Za-z]+
```

字母 + 数字
```regex
[A-Za-z0-9]+
```

Frame / frame / frm + 数字
```regex
([Ff]rame|frm)([0-9]+)
```

文件名（不含路径）
```regex
[^/]+$
```

扩展 Frame 写法
```regex
([Ff]rame|frm)[_-]?([0-9]+)
```

匹配：
```
Frame12
Frame_12
Frame-12
frm3
```

---

**典型完整示例**

```bash
file="/path/to/frame_003/output.yuv"

if [[ "$file" =~ ([Ff]rame|frm)[_-]?([0-9]+) ]]; then
    echo "完整匹配: ${BASH_REMATCH[0]}"
    echo "类型: ${BASH_REMATCH[1]}"
    echo "编号: ${BASH_REMATCH[2]}"
fi
```

输出：
```
完整匹配: frame_003
类型: frame
编号: 003
```

## 与 grep 正则的区别

| 项目         | Bash         | grep      |
| ------------ | ------------ | --------- |
| 使用位置     | `[[ =~ ]]`   | 命令      |
| 是否自动捕获 | ✅ 是        | ❌ 否     |
| 变量引用     | 直接变量     | 需要 echo |
| 引号规则     | ❌ 不加      | ✅ 常加   |

---

## Bash 正则不支持的内容

不支持：
* `\d`
* `\w`
* `\s`
* 非贪婪 `*?`
* lookahead / lookbehind

用替代写法：

| 想要     | 写法           |
| -------- | -------------- |
| 数字     | `[0-9]`        |
| 字母数字 | `[A-Za-z0-9_]` |
| 空白     | `[[:space:]]`  |

---

## 最佳实践

1. 正则 **永远不加引号**
2. 用 `()` 才能拿到 `BASH_REMATCH`
3. 能用 `[[ ]]` 就不用 `[ ]`
4. 路径匹配优先用字符串截取，复杂再用正则
5. 匹配完立刻用 `BASH_REMATCH`，别跨语句

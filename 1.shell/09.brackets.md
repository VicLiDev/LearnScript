# Shell脚本中常用到的括号整理

## 双引号 " "

双引号常用于包含一组字符串，在双引号中，除了 `$`、`\`、`(反引号)` 有特殊含义外，
其余字符（如IFS、换行符、回车符等）没有特殊含义。
```shell
$ a=3
$ echo "$a"
```
输出结果为 3，在双引号中 $ 符仍有特殊含义。

## 单引号 ' '

单引号的功能与双引号类似，不过单引号中的所有字符都没有特殊含义：
```shell
$ a=3
$ echo '$a'
```
输出结果为 $a，可见在单引号中 $ 符是不起作用的。

## 反引号 ` `
反引号的功能是命令替换，在反引号中的内容通常是命令行，程序会优先执行反引号中的
内容，并使用运行结果替换掉反引号处的内容。举个例子：
```shell
$ echo `date`
```
这行命令会先执行反引号中的命令 date，然后用 echo 命令打印出 date 命令的结果
（与直接使用 date 命令效果一样）。再举个例子：
```shell
#!/bin/bash

a=3
b=5
c=`expr $a \* $b`

echo $c

exit 0
```

## $ + 小括号 $( )
`$(...)` 的作用与反引号一样，也是命令替换：
```shell
#!/bin/bash

for file in $(ls /)
do
    echo $file
done

exit 0
```
这个脚本使用 for 循环打印根目录下所有文件的文件名，使用`$(ls /)`先获得根目录下的
所有文件，并将其作为参数列表传给 for 结构。

## $ + 双小括号 $(( ))
`$(( ))` 的功能是进行算术运算，括号中的内容为数学表达式，使用 `$(( ))` 可以求数学
表达式的值：
```shell
#！/bin/bash

a=3
b=5
c=$(($a * $b))
echo $c

exit 0
```
上述脚本的输出结果为 15。

使用 `$(( ))` 进行数学运算时，不需要担心乘号等运算符被 shell 误解为其他含义，因为
它们都在括号内。

## $ + 中括号 $[ ]

`$[ ]` 的功能与 `$(( ))` 一样，都是用于算术运算。

## $ + 大括号 ${ }

### 变量替换

`${ }` 的功能是变量替换，类似于 `$`，但比 `$` 的替换范围更精准：

```bash
a=3; b=5
echo $ab       # 空，ab 被当作一个变量
echo ${a}b     # 3b，明确取 a 的值再拼接 b
```

### 几种特殊的替换结构

这四种替换结构用于根据变量是否为空，来决定最终的结果：
```bash
${var:-word}   # var 为空 → 返回 word；                      var 不为空 → 返回 var 的值
${var:=word}   # var 为空 → 返回 word，同时把 word 赋给 var；var 不为空 → 返回 var 的值
${var:+word}   # var 为空 → 返回空；                         var 不为空 → 返回 word
${var:?word}   # var 为空 → 报错退出，输出 word；            var 不为空 → 返回 var 的值
```

记忆技巧：注意符号的方向——
- `:` 后面跟着 `-`、`=`、`+`、`?` 四种操作符，含义各不相同。
- `-` 和 `=` 是"为空时兜底"的意思（var 缺值时才用 word 补上），`=` 多了一步"顺便赋值"。
- `+` 是"有值时替换"的意思，和前两个逻辑正好相反。
- `?` 是"为空时报错"，常用于检查必要变量是否已设置。
> 注意：四种结构中，只有 `${var:=word}` 会修改 var 本身的值，其余三种都只产生替换结果，
> 不会改变 var。

下面逐个举例说明：

1. `${var:-word}` —— 为空时给个默认值
```shell
$ name=""
$ echo ${name:-guest}
guest                          # name 为空，返回默认值 "guest"
$ echo $name
                               # name 仍然是空，值没有被改变

$ name="alice"
$ echo ${name:-guest}
alice                          # name 不为空，返回 name 自身的值
```

2. `${var:=word}` —— 为空时赋默认值（和 `-` 类似，但会修改 var 本身）
```shell
$ name=""
$ echo ${name:=guest}
guest                          # name 为空，返回 "guest"，同时 name 被赋值为 "guest"
$ echo $name
guest                          # name 已经被改写了

$ name="alice"
$ echo ${name:=guest}
alice                          # name 不为空，不做任何修改，返回 name 自身的值
```

3. `${var:+word}` —— 不为空时替换（和 `-` 逻辑相反）
```shell
$ name="alice"
$ echo ${name:+hello}
hello                          # name 不为空，返回 "hello"

$ name=""
$ echo ${name:+hello}
                               # name 为空，什么都不返回（返回空）
```

4. `${var:?word}` —— 为空时报错退出（常用于检查必须设置的变量）
```shell
$ name=""
$ echo ${name:?请设置 name 变量}
bash: name: 请设置 name 变量    # name 为空，将 word 输出到 stderr，脚本退出

$ name="alice"
$ echo ${name:?请设置 name 变量}
alice                          # name 不为空，正常返回值
```

> 补充：word 不一定是固定的字符串，也可以是另一个变量或命令的输出，例如 `${var:-$HOME}` 或 `${var:-$(date)}`。

## 小括号 ( )

小括号有两种用途：数组初始化和命令组（子 shell）。

### 数组初始化

```bash
array1=(1 2 3 4 5)
array2=(one two three four five)
```

通过 `${array[index]}` 访问元素，索引从 0 开始：

```bash
$ echo ${array1[2]}        # 取第 3 个元素
$ 3

$ echo ${array2[0]}        # 取第 1 个元素
$ one

$ echo ${array2[*]}        # 输出整个数组
$ one two three four five
```

> 注意：数组变量的可移植性并不好，在 shell 编程中使用得不多。

### 命令组（子 shell）

`()` 将多条命令组成一个整体，在**子 shell** 中顺序执行。子 shell 中的变量修改、`cd`、
`exit` 等都不会影响当前 shell：

```bash
# 变量隔离
a=1; (a=2; echo "子 shell: a=$a"); echo "当前 shell: a=$a"
# 子 shell: a=2
# 当前 shell: a=1

# cd 隔离
pwd              # /home/user
(cd /tmp; pwd)   # /tmp
pwd              # /home/user

# exit 隔离：只退出子 shell，不影响当前 shell
(exit 0; echo "不会执行")
echo "当前 shell 仍在运行"
```

与 `{ }` 的区别：`{ }` 也在当前 shell 中组成命令组，但变量修改、`cd` 等会影响当前 shell，
且语法上最后一个命令后必须有 `;`，左括号前必须有空格：

```bash
# () —— 隔离环境
a=1; (a=2); echo $a    # 输出 1

# {} —— 共享环境
a=1; { a=2; }; echo $a  # 输出 2
```

### 行内变量赋值

`VAR=value command` 写法中，`VAR=value` 只对 `command` 生效，不会泄露到当前 shell：
```bash
MY_VAR=hello
MY_VAR=world echo $MY_VAR    # 输出 hello（$MY_VAR 在赋值前就展开了）
MY_VAR=world printenv MY_VAR  # 输出 world（printenv 看到了环境变量）
echo $MY_VAR                  # 输出 hello（当前 shell 未变）
```

行内赋值无论加不加 `()` 都不会泄露变量，加括号的区别在于命令在子 shell 中执行：
```bash
# 不加括号：cd 会影响当前 shell
MY_VAR=hello cd /tmp && pwd   # 当前 shell 已切换到 /tmp

# 加括号：cd 只影响子 shell
(MY_VAR=hello cd /tmp)
pwd  # 仍在原来的目录
```

实际应用举例：临时修改环境变量运行程序，用完自动恢复
```bash
# 临时指定 ADB 路径启动 scrcpy
(ADB="/path/to/custom_adb" scrcpy)

# 临时指定 GOBIN 编译安装
(GOBIN="$HOME/local/bin" go install ./...)

# 临时切换语言环境
(LC_ALL=C sort file.txt | uniq)
```

## 双小括号 (( ))
双小括号命令允许在比较过程中使用高级数学表达式：
```shell
(( expression ))
```
其中，expression 可以是任意的数学赋值或表达式。相比 test 命令只能在比较中使用简单
的算术操作，双小括号命令提供了更多的数学符号，可以在双小括号中进行各种逻辑运算、
数学运算，也支持更多的运算符（如 ++、-- 等）。

常使用的双小括号来在 for 循环中实现 C 语言风格的迭代：
```shell
#!/bin/bash

for ((i = 0; i < 10; i++))
do
    echo -n "$i "
done
echo ""

exit 0
```
如果不使用双括号, 则为for i in `seq 0 4` 或者 `for i in {0 1 2 3 4}`。再如可以
直接使用 `if (($i<5))`, 如果不使用双括号, 则为`if [ $i -lt 5 ]`。

## 中括号 [ ]
单个的中括号的功能与 test 命令一样，都是用作条件测试。
```shell
#!/bin/bash

read -p "please enter a number: " num

if [ $num -gt 10 ]; then
    echo "num > 10"
else
    echo "num <= 10"
fi
```

## 双中括号 [[ ]]
双中括号提供了针对字符串比较的高级特性，使用双中括号 [[ ]] 进行字符串比较时，可以
把右边的项看做一个模式，故而可以在 [[ ]] 中使用正则表达式：
```shell
#！/bin/bash

if [[ hello == hell* ]]; then
    echo "equal"
else
    echo "unequal"
fi

exit 0
```

## 大括号 { }
大括号用于括起一个语句块。如果需要在某些只能使用单个语句的地方（如AND、OR列表中）
使用多条语句，则可以用大括号将这多条语句括起来构造一个语句块。
单大括号，{ cmd1;cmd2;cmd3;} 在当前shell顺序执行命令cmd1,cmd2,cmd3, 各命令之间
用分号隔开, 最后一个命令后必须有分号, 第一条命令和左括号之间必须用空格隔开。

总结：

| 功能                                     | 符号             |
|------------------------------------------|------------------|
| 引用字符串，字符串中部分特殊符号有意义   | 双引号""         |
| 引用字符串，字符串中特殊符号全都没有意义 | 单引号''         |
| 命令替换                                 | (反引号)，`$()`  |
| 算数运算                                 | $(())、$[]、(()) |
| 变量替换                                 | ${}              |
| 数组初始化                               | ()               |
| 条件测试                                 | []               |
| 字符串比较                               | [[ ]]            |
| 括起一个语句块                           | { }              |


对{}和()而言, 括号中的重定向符只影响该条命令， 而括号外的重定向符影响到括号中的所有命令。
